# Learnings

Este diretorio armazena aprendizados e ajustes feitos durante a implementacao do Design System da NXZ.

## Como usar

Ao implementar componentes e encontrar situacoes onde os tokens ou patterns precisaram ser ajustados, documente aqui:

- **O que aconteceu:** Descreva o problema ou descoberta
- **O que foi ajustado:** Qual token, pattern ou regra mudou
- **Por que:** Justificativa da mudanca

## Formato sugerido

Crie arquivos markdown com a data no nome: `YYYY-MM-DD-descricao.md`

Exemplo: `2026-02-21-ajuste-contraste-primary.md`
